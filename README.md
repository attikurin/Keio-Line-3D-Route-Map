# 京王電鉄 3D 路線図ビューア

京王電鉄の全路線を、**線路だけ**（背景なし）で 3D 表現したインタラクティブ路線図です。
高低差・直線/カーブの線形・駅・車庫/留置線を再現し、電車を実際に走らせて 360 度から眺められます。

> **全駅・実ダイヤ準拠版**：京王線は新宿〜京王八王子の**全32駅**を実座標で完全再現し、停車パターン（各停／快速／区間急行／急行／特急／京王ライナー）を京王電鉄の標準的な平日ダイヤに準拠させています。東府中（競馬場線分岐）・中河原・百草園など中間駅もすべて収録。

## 🎯 プロジェクトの目的・主な特徴
- **線路だけで表現**：背景オブジェクトを置かず、無地空間に線路が浮かぶ表現。
- **高低差の再現**：実際の**海抜標高**に基づいて各駅の高さを設定。渋谷(谷地)→吉祥寺(武蔵野台地)の上り、新宿→八王子の台地・多摩川低地・丘陵の起伏、高尾線の山登り(高尾山口が最高)などを Y 座標で再現。高架は構造分の桁下高さ（脚付き）、地下はトンネル深さを加味。神泉(地下)より渋谷(高架ホーム・かつ谷地で実標高は低め)など、構造と実標高の関係も整合。駅情報パネルに概算海抜を表示。
- **実座標ベースの駅位置・駅間距離**：全駅の緯度経度を実際の位置（小数第6位）に基づいて設定。`geo()` で基準点(新宿)からの相対メートルに変換するため、**駅と駅の距離・方角・並びが実際の比率**で表現される（東西に長い京王線、放射状に伸びる相模原線・高尾線、武蔵野台地を横断する井の頭線などの全体像が正確）。駅間の中間制御点(via)で実際の線形カーブも再現。
- **実際の線路数を再現**：複線（上り線・下り線が分離）／単線（競馬場線・動物園線）を区別。駅構内では線路間隔が外側へ広がり、上り・下り・待避線が重ならないよう配置。待避駅は本線＋待避線の計4線＋ホームを表現。
- **可変オフセットの線形**：走行曲線を弧長基準の可変オフセットで生成し、駅で線路が滑らかに膨らむ／待避線が分岐する。線路がつながったまま列車がスムーズに移動。
- **すれ違い・追い越し**：上り列車と下り列車は別々の線路を走るため自然にすれ違う。時刻表モードでは各停が**実際に追い越しが発生する待避駅でのみ**待避線曲線に沿ってレールどおりに入線・退避し、特急・急行が本線を追い越す。無駄な待避動作をしないよう、優等列車が同駅を通過するタイミングをダイヤから自動検出し、追い越しがある駅だけ待避線へ入線する。
- **駅をクリックで詳細情報**：駅をクリックすると、停車する種別／通過する種別、線路構成（複線/単線）、構造（高架/地上/地下）、待避可否、隣の駅、接続路線、そして**発車時刻表**（上り・下り方面別、ダイヤから自動生成）を表示。
- **駅・車庫・留置線**：駅ホーム＋路線色マーカー、車庫には複数の留置線と留置車両を配置。
- **電車を走らせる**：各路線に1両編成が走行し、進行方向を向いて線路に沿って動く。端で自動折り返し。
- **2つの運行モード**：
  - **通常運転モード**：等速で全路線を行き来（速度調整可）。
  - **時刻表ダイヤモード**：実際の標準ダイヤ（駅間所要時間・停車時間・運行間隔・種別ごとの通過駅）を再現し、仮想時計に沿って複数列車が発車・走行・停車・折返しする。
- **360 度ビュー**：マウス/タッチで回転・ズーム・パン。

## ✅ 実装済み機能
- Three.js による 3D シーン（背景透過＝線路だけ）
- OrbitControls による 360 度回転・ズーム・パン
- 全 7 路線の描画（全駅収録）
  - **京王線（全32駅）** / 京王相模原線（全12駅） / 京王高尾線（全7駅） / 京王新線（全4駅） / 井の頭線（全17駅） / 競馬場線（2駅） / 動物園線（2駅）
- 路線ごとの色分け・2本レール＋路線色チューブ表現
- **複線/単線の再現**：複線は上り線・下り線を分離して描画、単線は1本を共用
- **待避駅**（京王線：笹塚・桜上水・八幡山・千歳烏山・つつじヶ丘・**調布**・東府中・府中・高幡不動・北野／相模原線：若葉台・京王多摩センター／高尾線：めじろ台／井の頭線：永福町 等）に本線＋待避線（計4線）・渡り線・島式ホームを表現
- 駅は線路の進行方向に合わせて回転配置し、ホーム・線路が線形と一致
- **すれ違い**（複線で上下が別レーン・駅で間隔拡大）と**追い越し**（時刻表モードで、追い越しが実際に発生する駅でのみ各停が待避線へレール沿いに入線、優等が本線を通過）
- 高架駅の脚、地下区間（負の高さ）の高低差表現
- 駅名ラベル（常にカメラを向く Sprite）
- 車庫・留置線（若葉台車両基地、富士見ヶ丘検車区 など）
- 電車の走行アニメーション（速度調整・停止/再開）
- UI：路線の表示/非表示、路線へカメラを寄せる、ラベル表示切替、全体ビューにリセット
- レスポンシブ対応（モバイルはパネル開閉ボタン）
- **種別ごとの色表示**（時刻表・停車チップ）：各停/快速/区急/急行/特急/京王ライナー をそれぞれ固有色で表示
- **時刻表モードの列車プール拡張**：全32駅化で同時走行本数が増えるため、京王線の列車メッシュプールを 56 本に拡張（相模原線18・高尾線16・井の頭線22 など路線別に確保）し、ピーク時でも全便が表示されるようにした
- **駅名ラベルのLOD／デクラッタ**：ズーム距離に応じて主要駅→優等停車駅→各停駅と段階表示し、密集による文字つぶれを防止

## 🗂 ファイル構成・機能エントリ
```

css/style.css   スタイル（背景＝無地、線路だけが浮く表現）

js/app.js       Three.js 本体（描画・電車走行・カメラ）→ window.KEIO_APP を公開
```

- 公開 API（JS）：`window.KEIO_APP`
  - `toggleLine(id, visible)` / `setRunning(bool)` / `setSpeed(num)` / `setTrainsVisible(bool)`
  - `focusLine(id)` / `resetView()` / `toggleLabels(bool)` / `lineList()`
  - `setMode('normal'|'timetable')` / `setSimTime(sec)` / `addSimTime(d)` / `setSimRate(r)` / `onTimeUpdate(cb)`
  - `onStationClick(cb)` / `getStationInfo(lineId, stIdx)` … 駅情報（種別・構造・隣駅・接続・発車時刻表）を返す

## 📊 データモデル（js/data.js）
- `LINES[lineId]`
  - `name`, `color`(16進)
  - `stations[]`：`{ name, lat, lon, y(高さm), grade('elevated'|'ground'|'underground') }`
  - `via{駅名: [{lat,lon,y}...]}`：駅間カーブ用の中間制御点
  - `depots[]`：`{ name, lat, lon, tracks(留置線数), y }`
  - `branchFrom`：分岐元路線・駅（参考情報）
- `TIMETABLE[lineId]`（時刻表ダイヤモード用）
  - `segSec[]`：駅間の標準所要時間（秒、駅数−1個）
  - `dwellSec`：駅停車時間、`turnSec`：終点折返し時間
  - `headwaySec`：運行間隔、`firstSec`/`lastSec`：始発/終電（00:00からの秒）
  - `services[]`：種別ごとの停車パターン（`stopsAll` または `stops` で停車駅index、`headwaySec`で種別別間隔、`firstSec`/`lastSec`で種別固有の運転時間帯）

### 時刻表モードの仕組み（js/app.js）
1. 各駅の弧長パラメータ `u` を曲線サンプリングで算出（`computeStationU`）。
2. `TIMETABLE` から、種別・方向（上下）・発車時刻ごとに「便（trip）」のスケジュール（各停車駅の到着/発車時刻）を生成（`buildTrip` / `buildAllTrips`）。
3. **追い越し検出（2パス）**：まず延長停車なしで全便を仮生成し、各停の待避駅停車中に同方向の優等が当駅 `u` を通過するか（`tripTimeAtU`）を判定。実際に追い越しが起きる駅にだけ `overtaken` フラグを付け、その駅でのみ延長停車（待避）して本生成する。→ 追い越しがない駅では待避線に入らず無駄な動きをしない。
4. 仮想時計 `simSec` を `simRate` 倍速で進め、現在アクティブな便の位置を補間（イーズ付き）して列車メッシュ（プール）に割り当てる。待避線への入線（`sideBlend`）は `overtaken` の駅でのみ発生（`updateTimetable` / `tripPositionU`）。
